## 一、核心结论（先说重点）

1. **项目方向被外部数据强验证**：`免费工具站 + 广告(AdSense)` 是 Google 官方点名的、
   稳定靠广告赚钱的 3 类网站之一（与 niche 博客、论坛并列）。多个 2026 年案例显示
   单人用 AI 辅助做工具站即可月入 $3k–$6k，极端案例（计算器站）月入 $50k。
   → 我们的 `zero-cost-earn-deploy`（27 个免费工具 + SEO 指南 + 自动推广）踩在正确赛道上。

2. **但赛道已拥挤，必须做差异化**：V2EX 老帖子明确警告“别卷工具站了”，
   先发站已瓜分权重；纯堆同质量工具难突围。胜出靠：**更好的 UX / AI 差异化功能 / 长线积累域名权重**。

3. **第二收入流（零成本、可自主）：联盟客（Affiliate）**。多个**免费加入、无流量门槛**
   的联盟计划可直接挂到工具站：walubee（70% 分成）、GoogieHost（$0.25/免费注册）、
   Themefreex（40%）、WhoKnew（30%）。这能在 AdSense 审批前/之外先产生收入。

4. **推广渠道排名（数据驱动，见下）**：Reddit（r/SideProject 等）转化率最高，
   BetaList / AlternativeTo / SaaSHub / StartupBase 等目录能在“发布日流量高峰”之后
   持续带来发现量（SEO 长尾）。Product Hunt 只有发布日尖峰。

## 四、建议的下一步（按 ROI 排序，全部零成本/可自主）

1. **【高】接入联盟客第二收入流**（零成本、可自主）
   - 在工具站页脚/侧栏嵌入 walubee（70%）与 GoogieHost（$0.25/注册）联盟位。
   - 收益：AdSense 过审前即可产生第一笔可提现收入，直接推进“第一笔真实收入”目标。

2. **【高】补齐高需求工具品类**（纯 JS、可自主，沿用 build.py 模式）
   - 优先：PDF 类（合并/拆分/压缩——需引轻量库如 pdf-lib）、在线图片编辑、
     AI 配音/克隆入口、二维码生成、密码生成、JSON 格式化、Markdown 工具。
   - 每个新工具 = 新的 SEO 落地页 + 指南，扩大内容表面积。

3. **【中】目录提交扩大发现面**（半自动/一次性的）
   - 向 BetaList / AlternativeTo / SaaSHub / StartupBase 提交站点（免费队列）。
   - 并入 promote.py 或独立脚本，跟踪提交状态。

4. **【中】用 Agent-Reach 建“选题监听”**（可自主，长期）
   - 周期性跑 Exa `site:reddit.com <抱怨关键词>` + V2EX，提炼下一批工具选题。
   - 形成“抱怨→工具→流量→收入”的自动闭环输入。

5. **【低/需人工】X / Reddit / HN 发布**（转化最高但需账号）
   - X 自动发帖器已就绪（post_x.py），待用户一次性创建免费 X App 填入凭证。
   - Reddit/HN 需人工号，存在封号风险，建议用独立小号且低频。

## 五、检索方法论（可复用）

- Exa：`mcporter` 在 Windows 上必须用**托管 node 直接调用 dist/cli.js**（绕过 cmd.exe 对
  括号/空格的参数切割），且 `NODE_TLS_REJECT_UNAUTHORIZED` 不应被设成 0。封装见
  `~/.agent-reach/resource_search.py`。
- V2EX 公开 API **无搜索端点**，需用 Exa `site:v2ex.com` 代替站内搜索。
- Bilibili 搜索 API 需带 `Referer: https://www.bilibili.com`，且结果分多类型块
  （video/upuser…），需跨块扁平化收集。
- Jina Reader 匿名端点在本机（数据中心 IP）被 401 拦截 → 本环境不可用，改靠 Exa。
